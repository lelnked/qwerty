# Feature Specification: 我的课程包与生词本

**Feature Branch**: `001-course-package-vocabulary`
**Created**: 2026-04-30
**Status**: Draft
**Input**: User description: "我想复制一下 https://julebu.co 中我的课程包和生词本的相关功能。功能的源码地址在 /home/lanshuangping/personal/earthworm 中。"

## User Scenarios & Testing *(mandatory)*

### User Story 1 - 浏览并进入课程包学习 (Priority: P1)

学习者打开"我的课程包"页面，可以看到自己拥有的课程包以卡片网格形式展示，每张卡片显示封面图、标题、简介与免费/会员标签。点击任一课程包后进入课程列表页，看到该课程包下的所有章节（课程），每个课程显示标题、简介与已完成次数，点击课程即可进入对应的练习/打字界面。

**Why this priority**: 这是整个功能的核心入口，没有它生词本功能也失去使用场景。课程包浏览与进入练习构成了 MVP 的最小闭环，单独交付即可让用户开始有结构地学习。

**Independent Test**: 在登录态下访问"我的课程包"路由，验证可以看到至少一组课程包卡片；点击任一课程包后能进入对应详情页并看到课程列表；从课程列表点击任一课程能跳转到对应的练习页面，无需依赖生词本功能。

**Acceptance Scenarios**:

1. **Given** 用户已登录且账户中已绑定至少一个课程包, **When** 用户进入"我的课程包"页面, **Then** 系统以网格布局展示该用户全部课程包的卡片，每张卡片显示封面、标题、描述与是否免费的标识，且加载过程中显示加载状态。
2. **Given** 用户在课程包列表页, **When** 用户点击任一免费课程包卡片, **Then** 系统跳转至该课程包详情页，标题位于顶部，下方以网格形式列出该课程包下所有课程，每张课程卡片包含标题、描述与已完成次数。
3. **Given** 用户在课程包详情页, **When** 用户点击任一课程卡片, **Then** 系统记录该课程包下的"当前活跃课程"并跳转到该课程的练习/打字页面。
4. **Given** 用户进入一个标记为"会员专享"的课程包, **When** 当前用户尚不是会员, **Then** 系统拒绝进入并以可见提示告知"该课程包需要会员权限"。

---

### User Story 2 - 收藏与管理生词本 (Priority: P1)

学习者在练习过程中遇到已经掌握或想要单独留存的单词/句子时，可以一键将其加入"生词本"。在"生词本"页面，用户可以看到全部已收藏条目（含英文内容、加入时间），通过搜索框模糊检索，并可删除不再需要的条目。系统在练习界面对已加入生词本的内容进行标记，避免重复学习。

**Why this priority**: 生词本是学习闭环中"沉淀"环节的关键，与 P1 的课程包并列同为核心；少了它，课程包学习无法形成"练习—收藏—回顾"的完整路径。可独立于其他高级功能交付。

**Independent Test**: 在任意练习页面触发"加入生词本"操作，进入"生词本"页面验证条目出现且按时间倒序排列；输入关键字验证模糊搜索结果；点击删除按钮验证条目被移除且页面计数刷新；返回练习页面验证已收藏的内容被标记/跳过。

**Acceptance Scenarios**:

1. **Given** 用户在练习页面正面对一个尚未收藏的单词/句子, **When** 用户触发"加入生词本"操作（按钮或快捷键）, **Then** 系统将该条目持久化保存，并立即在界面给出反馈，且后续在该条目上重新出现时显示"已收藏"状态。
2. **Given** 用户已经收藏多个生词条目, **When** 用户进入"生词本"页面, **Then** 系统以列表形式展示全部条目，按收藏时间倒序排列，顶部显示总条目数，并提供搜索框。
3. **Given** 用户在生词本页面输入搜索关键字, **When** 关键字与某条目英文内容部分匹配, **Then** 系统通过模糊匹配实时筛选并仅展示命中的条目。
4. **Given** 用户在生词本页面点击某条目的删除按钮, **When** 用户确认删除, **Then** 系统从持久化存储中移除该条目，列表与总数立即更新。
5. **Given** 用户尝试将同一条目重复加入生词本, **When** 系统检测到该英文内容（不区分大小写）已存在, **Then** 系统不创建重复条目并向用户给出"已在生词本中"的反馈。

### User Story 3 - 在练习中跳过/标注已收藏内容 (Priority: P2)

当学习者在课程练习中遇到已经收藏到生词本的条目时，系统能识别并按用户期望的方式处理（默认跳过或显著标记），使生词本真正起到"屏蔽已掌握内容"的作用。

**Why this priority**: 提升学习效率，但即便没有此项，P1 的两个故事仍能独立可用。属于体验优化层。

**Independent Test**: 提前向生词本中加入若干条目，再进入对应课程的练习页面，验证这些条目按设定方式被跳过或标注，且全章节练习的完成度计算考虑了被跳过的条目。

**Acceptance Scenarios**:

1. **Given** 当前课程章节中存在已被加入生词本的条目, **When** 用户进入该章节的练习, **Then** 系统按已配置策略跳过或显著标注这些条目，且不影响其他正常条目的练习流程。

---

### Edge Cases

- 当用户尚未拥有任何课程包时，"我的课程包"页面显示友好的空状态而非空白屏幕。
- 当课程包接口返回失败（网络/鉴权异常）时，页面显示错误状态并提供重试入口，而非永久停留在加载态。
- 当生词本中条目数量很大（例如超过 1000 条）时，列表仍然保持可滑动浏览且搜索响应在用户感知上保持流畅。
- 当用户处于离线状态时，已加载到本地的生词本应仍可浏览；新加入或删除的操作在恢复网络后能与服务端同步，且不丢失数据。
- 当未登录用户访问"我的课程包"或"生词本"页面时，系统引导其先登录，而非显示空数据。
- 当某课程包被标记为会员专享而用户非会员时，系统给出明确提示而不允许进入，但仍能在列表中查看其元信息。
- 当用户在练习中从生词本删除某条目后，再次进入练习时该条目应恢复为正常练习状态。

## Requirements *(mandatory)*

### Functional Requirements

- **FR-001**: 系统 MUST 提供"我的课程包"列表入口，仅展示当前登录用户拥有访问权限的课程包。
- **FR-002**: 系统 MUST 在课程包卡片上展示封面图、标题、简介与"免费/会员"标识。
- **FR-003**: 系统 MUST 在用户点击免费课程包时直接进入其课程列表，在用户点击会员专享课程包但身份不符时拒绝进入并明确提示原因。
- **FR-004**: 系统 MUST 在课程包详情页展示该课程包下全部课程，并对每门课程显示标题、简介与"已完成次数"。
- **FR-005**: 系统 MUST 记录用户在每个课程包下"最近选中"的课程，用于后续 Header/Sidebar 等位置的快速进入。
- **FR-006**: 系统 MUST 允许用户在练习界面通过按钮或键盘快捷键将当前条目加入生词本，并对操作结果给出即时反馈。
- **FR-007**: 系统 MUST 在重复加入同一英文内容（忽略大小写）时不创建重复条目，并提示用户该条目已存在。
- **FR-008**: 系统 MUST 提供独立的"生词本"页面，按收藏时间倒序展示全部条目，顶部显示总数，并提供搜索框。
- **FR-009**: 系统 MUST 在生词本页面支持基于英文内容的模糊搜索，并随用户输入实时筛选。
- **FR-010**: 用户 MUST 能在生词本页面单条删除条目，删除后列表与总数立刻刷新。
- **FR-011**: 系统 MUST 持久化保存生词本数据，使用户在不同会话/设备（在登录态下）能看到一致的数据。
- **FR-012**: 系统 MUST 在练习流程中识别已加入生词本的条目，并按既定策略（默认跳过）处理，避免重复练习。
- **FR-013**: 系统 MUST 在课程包列表/详情、生词本数据加载失败时呈现可重试的错误状态，而不是空白或永久加载。
- **FR-014**: 系统 MUST 在未登录访问"我的课程包"或"生词本"页面时引导用户登录。
- **FR-015**: 系统 MUST 在网络异常时对生词本的添加/删除操作做乐观更新与回滚，避免操作丢失或显示与实际不一致。

### Key Entities *(include if feature involves data)*

- **课程包 (CoursePack)**: 一组结构化课程的集合，包含 id、标题、简介、封面、是否免费/会员专享、所属课程列表。
- **课程 (Course)**: 隶属于某个课程包的单个练习单元，包含 id、标题、简介、所属课程包 id、当前用户已完成次数。
- **课程进度 (CourseHistory)**: 记录用户在某课程包下的练习历史，关键字段为课程 id、完成次数、最近练习时间，用于回填课程卡片上的完成数。
- **生词条目 (VocabularyItem)**: 用户收藏的英文内容记录，包含 id、英文内容、（可选）翻译/释义、收藏时间，与登录用户绑定。
- **当前活跃课程映射 (ActiveCourseMap)**: 每个课程包到该用户最近一次进入的课程的映射，用于侧边栏/Header 的快速跳转。

## Success Criteria *(mandatory)*

### Measurable Outcomes

- **SC-001**: 用户从"我的课程包"页面进入到具体课程的练习页面，3 次点击以内即可完成。
- **SC-002**: 课程包列表与详情页在常规网络条件下首屏可交互时间不超过 2 秒。
- **SC-003**: 在练习中通过快捷键加入生词本的操作，用户在 1 秒内可看到反馈。
- **SC-004**: 生词本页面在条目数达到 1000 条时，搜索输入到结果刷新的响应在用户感知上保持流畅（无明显卡顿）。
- **SC-005**: 90% 的新用户在首次进入"我的课程包"页面后能够独立完成"选择课程包 → 进入课程 → 开始练习"的路径，无需额外指引。
- **SC-006**: 重复加入生词本、对会员课程包的非会员访问等异常路径，100% 给出明确文案，不出现静默失败。

## Assumptions

- 目标载体为本仓库 qwerty（React 18 + Vite + Jotai + Dexie.js + Tailwind），不是直接照搬 earthworm 的 Nuxt + Pinia 实现；UI 与交互对齐 julebu.co，技术实现遵循 qwerty 既有规范。
- "我的课程包"与"生词本"均为登录用户功能；具体的鉴权/账号体系沿用 qwerty 当前已有的用户能力，不在本次特性内重新设计。
- 生词本数据的持久化默认包含两层：本地（沿用 Dexie.js 的 `newWords` 表，以保证离线可用）与服务端（与 julebu.co 行为对齐），具体同步细节在 plan 阶段决定。
- 课程包数据由后端提供（与 julebu.co 一致的接口形态，参考 earthworm `/course-pack` 路由），qwerty 端只负责消费。
- "会员/免费"判定逻辑与现有会员体系一致；本特性只负责调用判定结果，不引入新的付费流程。
- 移动端界面沿用 qwerty 现有的响应式策略（`/mobile` 路由）；本特性首要面向桌面端，移动端做最小可用适配。
- 练习中"已收藏即跳过"为默认策略；是否允许用户在设置中改为"标记但不跳过"留待后续迭代。
